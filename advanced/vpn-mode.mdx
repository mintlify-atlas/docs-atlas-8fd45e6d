---
title: "VPN mode: rootless global proxy using Android VPN API"
description: "Use Android's built-in VPN API and tun2socks to transparently route all device traffic through your SOCKS5 or HTTP proxy without root."
---

VPN mode lets you use ProxyDroid on any Android device — no root required. Instead of rewriting kernel routing tables, it uses Android's built-in `VpnService` API to create a virtual network interface and tunnel all traffic through your configured proxy.

## How it works

When you enable VPN mode, ProxyDroid creates a `tun` interface (`10.0.0.1/24`, MTU 1500) and routes all traffic through it (`0.0.0.0/0`). A native `tun2socks` library reads packets from the tun file descriptor and forwards them to the SOCKS5 proxy server. DNS is resolved via Google's public resolvers (`8.8.8.8` and `8.8.4.4`).

```text
App traffic
    │
    ▼
tun interface (10.0.0.1)
    │
    ▼
tun2socks (native library)
    │
    ▼
SOCKS5 proxy (your server)
```

**HTTP proxy bridging.** tun2socks speaks SOCKS5 natively. If you configure an HTTP proxy, ProxyDroid starts a `LocalProxyServer` on port `1080` that speaks SOCKS5 locally and translates outbound connections to your HTTP proxy. tun2socks then targets `127.0.0.1:1080` instead of your proxy directly.

<Note>
VPN mode does not require root, but it cannot intercept UDP traffic unless your proxy supports UDP association. The iptables mode offers lower overhead on rooted devices because it redirects packets in the kernel without a userspace relay. Choose VPN mode when root is unavailable or when you want per-app control.
</Note>

## Enable VPN mode

ProxyDroid's VPN mode is activated through Android's system VPN settings. The `ProxyDroidVpnService` is a standard `VpnService` and must be started via Android's VPN framework.

<Steps>
  <Step title="Configure your proxy settings">
    Launch ProxyDroid and enter your proxy server's host, port, and proxy type (SOCKS5, SOCKS4, or HTTP) on the main settings screen.
  </Step>
  <Step title="Toggle the Running switch">
    Tap the **Running** toggle at the top of the settings screen. If your device lacks root access, ProxyDroid automatically falls back to `ProxyDroidVpnService` (VPN mode).
  </Step>
  <Step title="Accept the VPN consent dialog">
    The first time you enable VPN mode, Android displays a system consent dialog asking you to authorize the VPN connection. Tap **OK** to continue. This dialog is a mandatory Android security requirement.
  </Step>
  <Step title="Confirm VPN is active">
    A persistent notification appears confirming the VPN service is running. The VPN key icon appears in the Android status bar.
  </Step>
</Steps>

## Per-app routing

VPN mode supports granular routing so you can choose exactly which apps travel through the proxy.

| Mode | Behavior |
|------|----------|
| **Proxy apps** | Only listed apps are routed through the VPN (`addAllowedApplication`). All others bypass the proxy. |
| **Bypass apps** | All apps are routed through the VPN except the listed ones (`addDisallowedApplication`). |

Open **App Manager** from the main screen to select apps. System apps with a UID below 10,000 are excluded from the list.

## Always-on VPN

ProxyDroid declares `android.net.VpnService.SUPPORTS_ALWAYS_ON = true` in its manifest. This means Android can automatically restart the VPN after a reboot or if the service is killed, with no traffic leaking in between.

To enable it:

1. Go to **Android Settings → Network & internet → VPN**.
2. Tap the gear icon next to ProxyDroid.
3. Enable **Always-on VPN** and optionally **Block connections without VPN**.
